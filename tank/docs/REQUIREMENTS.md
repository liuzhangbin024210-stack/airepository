# 坦克大战小游戏 — 需求文档

## 1. 项目概述

### 1.1 项目名称

基于 Python 的 2D 坦克大战单机小游戏（本地运行）。

### 1.2 目标用户

学习 Python / Pygame 的开发者，以及希望快速体验经典坦克玩法的休闲玩家。

### 1.3 项目目标

在单进程、单机环境下，实现可玩性完整的「玩家坦克 vs 敌方坦克 + 可破坏地形」核心循环，代码结构清晰，便于后续扩展（双人、关卡编辑器等）。

---

## 2. 功能需求

### 2.1 必须实现（MVP）

| 编号 | 需求 | 说明 |
|------|------|------|
| F-01 | 游戏窗口 | 固定分辨率窗口（建议 800×600 或 960×720），标题与基础帧率控制（如 60 FPS）。 |
| F-02 | 玩家坦克 | 键盘控制移动（四方向或八方向，以实现为准）、发射子弹；有生命值或一击必杀（文档推荐：3 条命或单条命 + 血量，实现阶段二选一）。 |
| F-03 | 敌方坦克 | 至少 1 种 AI：随机转向 + 朝玩家间歇射击；生成数量与重生规则在实现中可配置。 |
| F-04 | 子弹 | 直线飞行、与墙/坦克碰撞后消失；可对坦克造成伤害。 |
| F-05 | 地图与障碍 | 砖墙（可摧毁）、钢墙（不可摧毁）等至少两类地形；地图可用二维数组或文件描述。 |
| F-06 | 胜负条件 | 玩家全灭则游戏结束（失败）；消灭本关全部敌人或达成简单目标则胜利（首版可简化为「消灭所有敌方坦克」）。 |
| F-07 | UI 反馈 | 开始界面或按键开始、游戏结束提示、简单分数或击杀统计（可选）。 |

### 2.2 建议实现（增强）

| 编号 | 需求 | 说明 |
|------|------|------|
| F-08 | 道具 | 如加血、加速、子弹强化（可延后到 v1.1）。 |
| F-09 | 关卡 | 多关卡地图切换；关卡间过渡画面。 |
| F-10 | 音效 / 贴图 | 使用简单几何图形亦可；若有资源则替换为精灵图与音效。 |

### 2.3 暂不纳入范围

- 网络联机对战。
- 复杂寻路（A* 全图）；首版可用简单随机 + 朝向玩家射击即可。

## 3. 非功能需求

| 类别 | 要求 |
|------|------|
| 运行环境 | Python 3.10+；依赖通过 `requirements.txt` 固定（主库建议 Pygame 2.x）。 |
| 性能 | 在普通办公本上稳定 60 FPS（同屏对象数量在合理范围内）。 |
| 可维护性 | 模块划分：配置、精灵/实体、地图、游戏状态、主入口分离。 |
| 可测试性 | 核心碰撞与地图解析可抽函数，便于单元测试（可选）。 |

---

## 4. 输入与操作约定

- **移动**：方向键或 WASD（在实现中统一为一种并在 README 中说明）。
- **射击**：空格键；设置射击冷却，避免连续刷屏。
- **暂停 / 返回**：P 或 Esc（可选，列入开发计划）。

---

## 5. 数据与配置

- 地图：推荐使用字符矩阵或 JSON/TOML 描述格子类型（空、砖、钢、出生点等）。
- 常量：窗口大小、格子尺寸、坦克速度、子弹速度、冷却时间等集中放在 `config` 模块。

---

## 6. 验收标准（MVP）

1. 执行 `python -m tank` 或 `python main.py`（以最终实现为准）可启动游戏。
2. 玩家可移动、射击；子弹与墙、敌我坦克正确碰撞。
3. 至少 3 个敌方实例或分批生成，全部击毁后显示胜利；玩家被击中至生命耗尽显示失败。
4. 无明显逻辑错误（如穿墙、同阵营误伤若设计为关闭则需一致）。
5. `requirements.txt` 可复现安装依赖。

---

## 7. 风险与假设

- **假设**：用户使用支持 Pygame 的桌面系统（Windows / macOS / Linux）。
- **风险**：不同机器帧率差异 — 使用基于时间的移动（delta time）或固定逻辑步长缓解。
