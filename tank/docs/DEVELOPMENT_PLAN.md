# 坦克大战小游戏 — 开发计划

## 1. 技术选型

| 项 | 选择 | 备注 |
|----|------|------|
| 语言 | Python 3.10+ | 与需求文档一致 |
| 图形与输入 | Pygame 2.x | 2D 精灵、碰撞矩形、键盘事件成熟 |
| 项目布局 | 包结构 `tank/` | 可 `python -m tank` 启动 |

---

## 2. 目录结构（目标）

```
tank/
├── docs/
│   ├── REQUIREMENTS.md      # 需求文档
│   └── DEVELOPMENT_PLAN.md # 本文档
├── tank/                    # Python 包
│   ├── __init__.py
│   ├── __main__.py          # 入口：python -m tank
│   ├── config.py            # 常量与配置
│   ├── assets/              # 可选：图片、音效、关卡文件
│   ├── map_loader.py        # 地图解析
│   ├── entities.py          # Tank, Bullet, Wall 等精灵逻辑
│   ├── game.py              # 游戏循环、状态机、碰撞调度
│   └── ai.py                # 敌方行为（可选独立文件）
├── requirements.txt
└── README.md                # 安装与运行说明（实现阶段补充）
```

实现阶段可按实际需要微调文件名，但保持「配置 / 地图 / 实体 / 主循环」分离。

## 3. 阶段划分

### 阶段 0：工程脚手架（0.5 天）

- [ ] 创建 `requirements.txt`（`pygame` 等）。
- [ ] 建立包与 `__main__.py` 空窗口循环（清屏、事件退出、时钟）。
- [ ] 在 README 中写明创建虚拟环境与运行命令。

### 阶段 1：地图与静态碰撞（0.5–1 天）

- [ ] 定义格子类型枚举或常量。
- [ ] 实现地图加载（硬编码一关或从 `assets/levels/level01.txt` 读取）。
- [ ] 将地图渲染为矩形砖块/钢块；预留玩家与敌方出生点。

### 阶段 2：玩家坦克与子弹（1 天）

- [ ] `Tank` 类：位置、方向、速度、边界与墙碰撞（滑动或格对齐，二选一）。
- [ ] `Bullet` 类：发射、移动、出界销毁。
- [ ] 键盘输入绑定；射击冷却。
- [ ] 子弹与墙：砖墙扣血或销毁，钢墙阻挡。

### 阶段 3：敌方 AI 与阵营伤害（1 天）

- [ ] 敌方坦克生成与简单 AI（定时随机改向 + 概率射击）。
- [ ] 友军火力规则：玩家子弹不打玩家；敌方子弹不打敌方（可配置）。
- [ ] 坦克受击：生命或条数；死亡后移除或重生规则。

### 阶段 4：胜负与 UI（0.5 天）

- [ ] 游戏状态：`MENU` / `PLAYING` / `GAME_OVER` / `VICTORY`。
- [ ] 全灭敌人 → 胜利；玩家生命耗尽 → 失败。
- [ ] 简单文字提示（Pygame font）或开始画面。

### 阶段 5：打磨与可选增强（0.5–1 天）

- [ ] 调参：速度、冷却、敌人数、地图难度。
- [ ] （可选）第二关地图、基础音效。
- [ ] （可选）`pytest` 对 `map_loader`、碰撞工具函数做最小测试。

## 5. 关键设计决策（实现前确定）

1. **对齐方式**：坦克按「格子中心」对齐更贴近原作，但实现稍复杂；像素自由移动更易上手。首版推荐 **像素移动 + 与墙 AABB 碰撞**。
2. **地图坐标系**：左上角为原点，x 向右、y 向下，与 Pygame 一致。
3. **同帧多子弹**：玩家同时仅允许 1 发在场（降低难度）或允许多发 — 在 `config` 中开关。
